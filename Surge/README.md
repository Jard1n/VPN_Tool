# VPS 流量监控服务搭建指南

本指南将指导你如何在 VPS 上部署一个基于 Python 的流量与系统资源监控服务，并通过 7122 端口对外提供 API 数据。

## 1. 登录与获取权限

首先，通过 SSH 登录到你的 VPS。为了确保安装过程不受权限限制，切换到 root 用户：

```bash
sudo -i
```

## 2. 更新系统环境

在安装任何软件之前，建议先更新软件包列表并升级现有软件：

```bash
apt update && apt upgrade -y
```

## 3. 安装 Python3 与包管理器

如果你的系统中尚未安装 Python3，请执行以下命令进行安装：

```bash
sudo apt install python3 python3-pip -y
```

## 4. 安装依赖包

运行程序需要 `psutil` 库来获取系统信息。

### 标准安装：
```bash
pip3 install psutil
```

### 常见问题处理：
如果在执行上一条命令时遇到报错（如：*externally-managed-environment*），请尝试以下备选方案：

**方案 A：强制在系统环境下安装**
```bash
pip3 install psutil --break-system-packages
```

**方案 B：使用 APT 安装系统级包（推荐）**
```bash
apt update
apt install python3-psutil -y
```

> **提示**：后续运行程序时，如果提示 `ModuleNotFoundError: No module named xxxx`，请使用上述方法安装对应的 `xxxx` 依赖包。

---

## 5. 编写监控程序

创建并编辑监控脚本：

```bash
vim /root/servertraffic.py
```

将以下 Python 代码粘贴进去并保存（`:wq`）：

```python
import http.server
import json
import psutil
import time
import os
from datetime import datetime, timedelta
from urllib.parse import urlparse, parse_qs

STATS_FILE = '/root/traffic_stats.json'

def get_current_cycle_start(reset_day):
    """根据重置日期计算当前周期的起始日期"""
    now = datetime.now()
    try:
        # 如果今天的日期大于等于重置日期，说明本月已经重置过了
        if now.day >= reset_day:
            cycle_start = now.replace(day=reset_day, hour=0, minute=0, second=0, microsecond=0)
        else:
            # 说明现在还在上一个计费周期的末尾（跨月了）
            first_of_this_month = now.replace(day=1)
            last_month_end = first_of_this_month - timedelta(days=1)
            # 处理部分月份没有31号的情况，取最小值
            target_day = min(reset_day, last_month_end.day)
            cycle_start = last_month_end.replace(day=target_day, hour=0, minute=0, second=0, microsecond=0)
        return cycle_start.strftime("%Y-%m-%d")
    except Exception:
        return now.strftime("%Y-%m-01")

def get_traffic_stats(req_reset_day):
    net_io = psutil.net_io_counters()
    curr_sent = net_io.bytes_sent
    curr_recv = net_io.bytes_recv
    
    stats = {
        "cycle_id": "",
        "reset_day": req_reset_day, # 默认用请求传来的
        "total_sent": 0, "total_recv": 0,
        "last_hw_sent": curr_sent, "last_hw_recv": curr_recv
    }

    if os.path.exists(STATS_FILE):
        try:
            with open(STATS_FILE, 'r') as f:
                saved_stats = json.load(f)
                
                # 【重要修复】：如果请求没有传重置日(例如传入的None)，则使用上次保存的重置日
                if req_reset_day is None and "reset_day" in saved_stats:
                    stats["reset_day"] = saved_stats["reset_day"]
                elif req_reset_day is not None:
                    stats["reset_day"] = req_reset_day
                    
                # 重新计算此时的 cycle_id
                cycle_id = get_current_cycle_start(stats["reset_day"])
                
                # 只有周期 ID 一致才继承旧数据
                if saved_stats.get("cycle_id") == cycle_id:
                    stats = saved_stats
                    stats["reset_day"] = req_reset_day if req_reset_day is not None else saved_stats.get("reset_day", 1)
        except: pass

    # 无论如何重新计算一次当前应有的周期 ID
    current_cycle_id = get_current_cycle_start(stats["reset_day"])

    # 计算差值（考虑重启情况）
    diff_sent = curr_sent - stats["last_hw_sent"]
    diff_recv = curr_recv - stats["last_hw_recv"]
    if diff_sent < 0: diff_sent = curr_sent
    if diff_recv < 0: diff_recv = curr_recv
    
    stats["total_sent"] += diff_sent
    stats["total_recv"] += diff_recv
    stats["last_hw_sent"] = curr_sent
    stats["last_hw_recv"] = curr_recv
    stats["cycle_id"] = current_cycle_id

    with open(STATS_FILE, 'w') as f:
        json.dump(stats, f)
    return stats

class SimpleHTTPRequestHandler(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        query_components = parse_qs(urlparse(self.path).query)
        
        # 【重要修复】：如果没有传参数，传入 None 而不是默认的 1
        reset_day_str = query_components.get("reset_day")
        reset_day = int(reset_day_str[0]) if reset_day_str else None
        
        traffic = get_traffic_stats(reset_day)
        cpu = psutil.cpu_percent(interval=None)
        mem = psutil.virtual_memory().percent
        uptime = int(time.time() - psutil.boot_time())
        
        data = {
            "cpu_usage": cpu,
            "mem_usage": mem,
            "bytes_sent": traffic["total_sent"],
            "bytes_recv": traffic["total_recv"],
            "bytes_total": traffic["total_sent"] + traffic["total_recv"],
            "uptime": uptime,
            "cycle_start": traffic["cycle_id"],
            "last_time": datetime.now().strftime("%H:%M:%S")
        }
        
        self.send_response(200)
        self.send_header('Content-type', 'application/json')
        self.send_header('Access-Control-Allow-Origin', '*')
        self.end_headers()
        self.wfile.write(json.dumps(data).encode())

print("Monitor Server running on 7122 with Dynamic Reset Support...")
httpd = http.server.HTTPServer(('0.0.0.0', 7122), SimpleHTTPRequestHandler)
httpd.serve_forever()
```

> **注意**：请确保 VPS 防火墙已开放 **7122** 端口。

## 6. 配置 Systemd 系统服务

为了让程序在后台持续运行并开机自启，我们需要创建一个服务单元文件：

```bash
vim /etc/systemd/system/servertraffic.service
```

粘贴以下配置：

```ini
[Unit]
Description=Server Traffic Monitor
After=network.target

[Service]
Type=simple
WorkingDirectory=/root/
User=root
ExecStart=/usr/bin/python3 /root/servertraffic.py
Restart=always

[Install]
WantedBy=multi-user.target
```

---

## 7. 启动服务与管理

完成配置后，执行以下命令使服务生效：

### 启动并设置开机自启
```bash
sudo systemctl daemon-reload
sudo systemctl start servertraffic.service
sudo systemctl enable servertraffic.service
```

### 查看服务运行状态
```bash
sudo systemctl status servertraffic.service
```

### 常用管理命令
- **停止服务**: `sudo systemctl stop servertraffic.service`
- **重启服务**: `sudo systemctl restart servertraffic.service`
- **查看实时日志**: `journalctl -u servertraffic.service -f`

---

## 8. 使用说明

- **默认监控端口**: `7122`
- **API 接口**: `http://你的VPS_IP:7122/`
- **自定义流量重置日期**: 可以在 URL 后增加参数，例如 `http://IP:7122/?reset_day=15`（表示每月 15 号重置流量统计）。
